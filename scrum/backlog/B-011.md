## Descricao

Atualmente as diretivas `@ref(uri)` e `@include(uri)` aparecem como texto plano
em qualquer viewer Markdown (GitHub, VS Code, Obsidian). O objetivo e adotar uma
sintaxe que seja simultaneamente:

1. Parseavel pelo motor mdgraph
2. Renderizavel como link clicavel em qualquer renderer Markdown padrao

## Sintaxe nova (breaking change)

```markdown
[@ref: Titulo da Secao](caminho/arquivo.md#id)
[@include: Titulo da Secao](caminho/arquivo.md#id)
[@query: Descricao](expressao)
```

- O texto do link `@ref: Titulo` preserva o tipo da diretiva e um label legivel.
- O href do link `(uri)` contem a URI de destino — que ja e um caminho de arquivo valido.
- Em renderers Markdown padroes, o resultado e um link clicavel com label `@ref: Titulo`.

## Escopo tecnico

### 1. Atualizar regex em `directives.py`

Novo padrao (substitui `_DIRECTIVE_RE` atual):
```python
_DIRECTIVE_RE = re.compile(
    r'\[@(ref|include|query)(?::\s*([^\]]*))?\]\(([^)]+)\)'
)
```
- Grupo 1: tipo (`ref|include|query`)
- Grupo 2: label opcional (pode ser vazio)
- Grupo 3: URI de destino

### 2. Atualizar modelo `Directive` em `models.py`

Adicionar campo `label: str | None = None` para preservar o texto descritivo.

### 3. Atualizar todas as fixtures de teste

Arquivos em `tests/fixtures/` que usam `@ref(...)` ou `@include(...)`:
- `with_ref.md`
- `with_include.md`
- `multi_directives.md`
- `cycles/*.md` (todos os arquivos de ciclo)
- `examples/wiki/*.md` (exemplos)

### 4. Atualizar testes

- `tests/test_directives.py` — todos os casos
- `tests/test_cli_compose.py` — testes de @include
- `tests/test_examples.py` — smoke tests dos exemplos

### 5. Atualizar README e examples

- Secao "Sintaxe das Diretivas" no README.md

## Criterios de aceite

- [ ] `[@ref: label](uri)` e reconhecido como Directive tipo `ref`
- [ ] `[@include: label](uri)` e reconhecido como Directive tipo `include`
- [ ] Label e preservado no campo `Directive.label`
- [ ] URI e resolvida identicamente a antes
- [ ] Sintaxe antiga `@ref(uri)` NAO e mais reconhecida (breaking change deliberado)
- [ ] Todos os 116 testes existentes passam (apos migracao das fixtures)
- [ ] Novos testes cobrem: label vazio, label com espacos, label ausente (`[@ref](uri)`)
- [ ] Arquivos de exemplo renderizam como links no VS Code preview

## Decisao de design

- Breaking change deliberado — nao ha retrocompatibilidade.
- Motivacao: manter dois parsers em paralelo aumentaria complexidade sem beneficio
  (o projeto nao tem usuarios externos ainda).
- Migracao dos arquivos existentes e feita na mesma PR do item.

## Riscos

- **R1 (medio):** fixtures de ciclo sao muitas; migracao manual e propensa a erro.
  Mitigacao: script de sed one-liner para migrar todas as ocorrencias.
- **R2 (baixo):** renderers que nao suportam links relativos (ex: alguns wikis)
  podem nao resolver o href corretamente. Fora do escopo deste item.
