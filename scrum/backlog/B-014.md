## Description

Two goals in one item:

1. **Translate** all user-facing Markdown files to English.
2. **Expand `specification.md`** with the mathematical specification and

## Scope — Translation

- `specification.md` (translated AND expanded, see below)

## Scope — specification.md expansion

The current spec covers sections 1–8. The expanded spec must add:

### Section 9 — Semantic Memory Model

Formal reframing of `mdgraph` as a **deterministic graph-based semantic
memory layer** for AI agents. Covers:

- Comparison with RAG (vector), knowledge graphs, Obsidian-style Markdown
- The stability property of `file.md#id` URIs vs. heading titles
- YAML as a first-class semantic index (not just decoration)
- The distinction between "document composition" (human output) and
  "context materialization" (AI input)

### Section 10 — Extended Graph Operations (mathematical model)

For each new command, define:

- **Formal signature**: $f: G \times V \rightarrow R$ (inputs and return type)
- **Algorithm**: BFS/DFS variant, set operations, path-finding
- **Complexity**: time and space $O(?)$
- **Output schema**: JSON structure

Commands to specify:

| Command | Algorithm | Notes |
|---------|-----------|-------|
| `validate` | Full index scan + edge verification | O(V+E) |
| `context` | 1-hop neighborhood extraction | O(deg(v)) |
| `backlinks` | Direct lookup in `incoming_edges` | O(1) |
| `search` | Linear scan with field predicate | O(V) |
| `impact` | Reverse BFS on `incoming_edges` | O(V+E) |
| `neighbors` | Bidirectional BFS | O(V+E) |
| `explain` | All simple paths (DFS) | O(V! worst case) |
| `diff` | Parse historical revision + set diff | O(V+E) |
| `query` | Parsed predicate + linear scan | O(V) |
| `context-compose` | Bounded compose with token budget | O(V+E) |

### Section 11 — JSON Contract

Defines the canonical JSON schema for every command's `--json` output.
This is the API contract for AI agent tool-calling integration.
