# B-009 - Cache persistente do indice

## Descricao

Implementar a camada de cache descrita na spec section 7: serializar o `SectionIndex` em
`.mdgraph/index.json` e, em execucoes subsequentes, reprocessar apenas os arquivos cujo
hash tenha mudado.

## Escopo

- Apos `index_repository()`, serializar o SectionIndex para `.mdgraph/index.json`
  - Incluir hash (SHA256 ou mtime+size) de cada arquivo .md processado
- No inicio de `index_repository()`: verificar se `.mdgraph/index.json` existe
  - Se sim: carregar cache e comparar hashes dos arquivos atuais
  - Reprocessar apenas arquivos com hash diferente do cache
  - Arquivos removidos devem ter suas secoes expurgadas do index
- Flag `--no-cache` para forcar reprocessamento completo

## Criterios de aceitacao

- [x] Primeira execucao cria `.mdgraph/index.json` com hash dos arquivos
- [x] Segunda execucao sem mudancas retorna mesmo resultado (cache hit)
- [x] Modificar um arquivo .md forca reprocessamento apenas desse arquivo
- [x] Remover um arquivo .md expurga suas secoes do index
- [x] `--no-cache` ignora o cache e reprocessa tudo
- [x] Testes em `tests/test_cache.py` verificando comportamento incremental (13/13)

## Riscos

- Medio. Invalidacao de cache e expurgo de secoes de arquivos deletados exigem atencao
  para nao deixar URIs fantasmas no index.
