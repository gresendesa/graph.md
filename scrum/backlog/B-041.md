# B-041 - Web-based template packages (fetching template ZIPs from web URIs with checksum verification)

## Description

Extend the workspace initialization (`mdb init`) to support retrieving template packages from remote web URIs (e.g. GitHub releases or raw files) and verifying their integrity using a SHA256 checksum.

This ensures that remote templates are fetched securely and initialized reproducibly.

## Scope

- Support `mdb init --template <URL>` where URL starts with `http://` or `https://`.
- Introduce a `--checksum` option in `mdb init` (supporting format `sha256:<hex>` or just the `<hex>` string).
- For web-based templates, make `--checksum` mandatory to guarantee security and reproducibility, failing validation if missing.
- Fetch the template ZIP into a temporary file or local cache, verify its SHA256 checksum against the expected value, and proceed with standard unpacking and signature validation.
- Preserve all existing local template zip functionality.
- Cache downloaded templates inside `.mdb/cache/templates/` to avoid repeated remote downloads unless `--no-cache` or `--force` is specified.

## Out of scope

- Storing credentials or authenticating against private repositories.
- Unpacking non-zip remote formats.

## Criteria of acceptance

- [x] `mdb init` accepts remote URLs for templates when `--checksum` is provided.
- [x] Initializing with a remote URL fails if `--checksum` is not provided.
- [x] The downloaded template zip is verified against the provided checksum, and initialization fails on mismatch.
- [x] Successfully downloaded and verified templates are cached under `.mdb/cache/templates/`.
- [x] Re-running `mdb init` with the same URL uses the cached zip if present and matches the checksum, unless `--no-cache` is specified.
- [x] Automated tests cover successful web template downloads, missing checksum failures, mismatching checksum failures, and local template regressions.
