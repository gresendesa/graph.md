## Description

Implement `mdb pack` and `mdb init` to enable template-driven workspace initialization from checksum-signed Zip packages. This moves `mdb` towards being a methodology-agnostic workspace orchestrator for AI agents by allowing templated project scaffolding (like the Scrum memory files).

## Scope

- **Jinja2 dependency**: Add `jinja2` to `pyproject.toml` dependencies.
- **`mdb pack <directory> --output <filename.zip>`**:
  - Combines a template source directory (containing `manifest.yaml` and template/schema files) into a deterministic signed `.zip` package.
  - Automatically creates a `SIGNATURE.yaml` with deterministic SHA256 hashes of every file in the package (except itself) to guarantee integrity.
  - Files are written to the ZIP archive in alphabetical order, using a fixed epoch date (`1980-01-01 00:00:00`) to guarantee byte-for-byte identical ZIP generation.
- **`mdb init --template <package.zip>`**:
  - Initializes a new directory using a signed template package.
  - Performs Signature Verification against `SIGNATURE.yaml` in memory, raising an error if any file has been modified, added, or missing.
  - Resolves context variables declared in `manifest.yaml` (using defaults, prompting, or reading key-value inputs).
  - Renders Jinja2 templates under target names. Refuses to overwrite existing files unless `--force` is specified.
  - Refuses to initialize over existing memory files or config unless `--force` is specified.
  - Writes a configuration file `.mdb/config.yaml` tracking the project variables and initialized profile.
- **CLI commands registration**: Integrate the new commands in `mdbind/cli.py` using `typer`.
- **Documentation**: Update `specification.md`, `README.md` and CLI documentation to cover `mdb pack` and `mdb init`.
