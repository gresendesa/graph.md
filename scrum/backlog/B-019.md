## Description

Given a node URI, performs a reverse BFS (following incoming edges) to find
all nodes that would be affected by a change to the target node.
Separates direct and indirect impact.

This is the "blast radius" operation — the most valuable operation for
maintenance agents and change impact analysis.

## Syntax

```bash
mdgraph impact docs/auth.md#oauth --root docs/
mdgraph impact docs/auth.md#oauth --root docs/ --depth 3
mdgraph impact docs/auth.md#oauth --root docs/ --json
```

## Output (default)

```
Direct impact (depth 1):
  login.md#web-login
  api.md#authentication

Indirect impact (depth 2):
  billing.md#checkout
  reports.md#monthly

Total: 4 nodes affected
```

## Output (--json)

```json
{
  "uri": "auth.md#oauth",
  "direct": ["login.md#web-login", "api.md#authentication"],
  "indirect": ["billing.md#checkout", "reports.md#monthly"],
  "total": 4
}
```

## Acceptance criteria

- [ ] Direct impact = immediate backlinks (depth 1)
- [ ] Indirect impact = BFS beyond depth 1
- [ ] --depth N limits traversal depth
- [ ] --json produces structured output
- [ ] Cycles in backlink graph do not cause infinite loops
- [ ] Tests: node with no impact, direct only, multi-level
