# B-029 - JSON context fails with YAML date metadata

- Status: done
- PO Priority: 2 (high)
- Risk: 2 (medium)
- Owner: gresendesa
- Creation date: 2026-06-09
- Last updated: 2026-06-09

## Description

Running `mdb context <uri> --root . --json` fails when a section metadata block
contains date-like YAML values such as `created_at: 2026-06-08`.

Observed error:

```text
TypeError: Object of type date is not JSON serializable
```

The expected behavior is that `--json` emits valid JSON even when YAML metadata
contains date values. Date-like metadata must be serialized as strings, or
metadata must otherwise be normalized into JSON-safe values before output.

`mdb context-compose <uri> --root . --json` works on the same repository, so the
bug is expected to be isolated to the `context` JSON serialization path or to
metadata normalization used by that command.

## Implementation notes

- `src/mdbind/cli.py` now uses an internal `_json_dumps` helper with
  `default=str` for CLI JSON output.
- `mdb context --json` serializes YAML date metadata such as
  `created_at: 2026-06-08` as `"2026-06-08"`.
- The helper is used by CLI JSON outputs broadly to avoid the same metadata
  serialization issue in related commands such as `search` and `query`.
