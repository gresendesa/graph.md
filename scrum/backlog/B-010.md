## Descricao

Criar documentacao voltada a humanos e LLMs: um README.md completo na raiz do projeto
e um diretorio `examples/` com repositorios Markdown funcionais que demonstram
todos os recursos do motor.

O README deve ser estruturado de forma que:
- Um desenvolvedor humano consiga instalar, entender a arquitetura e usar a CLI sem ler o codigo.
- Um LLM com o README no contexto consiga responder perguntas sobre o projeto, gerar snippets
  corretos e entender o modelo mental do grafo documental.

## Escopo

### README.md (raiz)

Secoes obrigatorias (com ids de secao para o proprio motor poder indexar):

1. Visao geral — o que e o mdgraph, para que serve, diferencial
2. Instalacao — pre-requisitos, virtualenv, `pip install -e .`
3. Conceitos fundamentais — Secao, URI, Diretivas, Pipeline, Grafo
4. Referencia da CLI:
   - `mdgraph get <URI>` — fidelidade documental
   - `mdgraph tree <URI> [--refs]` — hierarquia visual
   - `mdgraph compose <URI> [--strict] [--deduplicate] [--json]` — materializacao
5. Sintaxe do bloco `section` — formato YAML, campo `id`, campos livres
6. Sintaxe das diretivas — `@ref`, `@include`, `@query` (reservado)
7. Comportamento de ciclos — deteccao silenciosa, grafo preservado
8. Cache incremental — `.mdgraph/index.json`, SHA-256, `--no-cache`
9. Arquitetura interna (resumida) — pipeline de 5 etapas, modelos Pydantic
10. Exemplos rapidos — snippets de uso com saida esperada

### examples/

- `examples/README.md` — introducao ao diretorio de exemplos
- `examples/wiki/` — repositorio simulado de wiki com:
  - `intro.md` — pagina raiz com @include e @ref
  - `conceitos/basico.md` — secao folha sem dependencias
  - `conceitos/avancado.md` — secao com @ref para basico
  - `guia/instalacao.md` — secao com @include de avancado
- Todas as secoes devem ter blocos `section` validos com `id`, `title` e campos extras

## Criterios de aceitacao

- [ ] `mdgraph get examples/wiki/intro.md#intro` retorna a secao corretamente
- [ ] `mdgraph tree examples/wiki/guia/instalacao.md#instalacao --root examples/wiki` exibe arvore
- [ ] `mdgraph compose examples/wiki/intro.md#intro --root examples/wiki` gera documento unificado
- [ ] README explica todos os 3 comandos com exemplos de saida
- [ ] README explica formato do bloco `section` e diretivas
- [ ] README e legivel por LLM sem ambiguidades de contexto
