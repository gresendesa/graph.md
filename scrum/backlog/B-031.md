# B-031 - `mdb diff` fails on historical Markdown parsing with PosixPath file_path

- Status: done
- PO Priority: 1 (critical)
- Risk: 2 (medium)
- Owner: gresendesa
- Creation date: 2026-06-11
- Last updated: 2026-06-11
- Dependencies: B-022

## Description

Running `mdb diff --root . --since HEAD --json` can fail when the command parses
historical Markdown files read from Git. The command exits with an internal
Pydantic validation traceback because `ParsedSection.file_path` receives a
`PosixPath` value instead of a string.

Observed error:

```text
ValidationError: 1 validation error for ParsedSection
file_path
  Input should be a valid string [type=string_type,
input_value=PosixPath('/path/to/repo/mdbind.md'),
input_type=PosixPath]
```

The expected behavior is that `mdb diff` returns a JSON structural diff with
added and removed sections and edges.

## Scope

- Reproduce the failure in a Git repository containing MDBind Markdown files.
- Add an automated regression test covering historical Markdown parsing in
  `mdb diff --json`.
- Fix the type mismatch in the historical parsing path by passing a string file
  path to `parse_text`.
- Preserve the existing `mdb diff --json` output contract.
- Ensure related `mdb diff` validation paths remain unchanged.

## Likely cause

Inside `src/mdbind/cli.py`, the `diff` command builds historical graph sections
with a `Path` object:

```python
abs_path = Path(git_root) / rel_path
sections = parse_text(content, abs_path)
```

`ParsedSection.file_path` expects a string, so the historical parsing path should
pass `str(abs_path)`:

```python
sections = parse_text(content, str(abs_path))
```

## Reproduction

From a Git repository containing MDBind Markdown files:

```bash
mdb diff --root . --since HEAD --json
```

## Implementation notes

- `src/mdbind/cli.py` now passes `str(abs_path)` to `parse_text` when building
  the historical graph used by `mdb diff`.
- The change is scoped to the Git historical parsing path and does not alter
  the `mdb diff --json` schema.
