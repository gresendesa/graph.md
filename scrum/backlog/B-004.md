## Descricao

Implementar a etapa 5 do pipeline (spec sections 2, 5 e 7): varrer todos os arquivos .md de um
diretorio, construir o `SectionIndex` completo em memoria e montar o `SectionGraph` com arestas
direcionais bidirecionais.

## Escopo

### index_repository(root_path) -> SectionGraph
- Descobrir recursivamente todos os arquivos `.md` no diretorio raiz
- Para cada arquivo: executar o pipeline B-002 + B-003 produzindo ParsedSections
- Registrar cada ParsedSection no `SectionIndex` usando sua URI como chave
- Detectar e reportar colisoes de URI `(arquivo, id)` duplicados

### Construcao do SectionGraph
- Para cada ParsedSection com diretivas, criar arestas:
  - `outgoing_edges[uri_origem].add(uri_destino)` para cada diretiva ref/include
  - `incoming_edges[uri_destino].add(uri_origem)` para indexacao de backlinks
- Indexacao bidirecional para busca em O(1)

## Criterios de aceitacao

- [x] `index_repository("./fixtures/")` retorna SectionGraph populado com todas as secoes
- [x] URIs duplicadas (mesmo arquivo, mesmo id) levantam erro com mensagem clara
- [x] `outgoing_edges` e `incoming_edges` sao preenchidos corretamente
- [x] Lookup `index.sections["arquivo.md#id"]` retorna ParsedSection correta
- [x] Testes em `tests/test_index.py` com repositorio de fixtures cobrindo os casos acima (11/11)

## Riscos

- Medio. Performance em repositorios grandes e tratamento de encoding de arquivos podem
  exigir atencao. URI como chave exige normalizacao de caminhos.
