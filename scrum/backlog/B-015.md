## Description

CLI command that validates the structural integrity of a repository.
Essential for CI/CD pipelines and as a pre-flight check before any agent operation.

## Syntax

```bash
mdgraph validate --root docs/
mdgraph validate --root docs/ --json
```

## Checks

1. **Duplicate IDs** — same `section:` value in the same file
2. **Broken references** — `@ref` or `@include` targeting a URI not in the index
3. **Missing payloads** — `.md` files with headings but no `yaml` block with `section:`
4. **Include cycles** — circular `@include` chains
5. **Schema inconsistencies** — reserved fields with wrong types (future)

## Output

Human-readable:
```
ERROR  docs/auth.md#oauth        broken ref -> security.md#legacy (not found)
WARN   docs/intro.md             heading "Overview" has no section block
OK     12 sections validated, 0 errors, 1 warning
```

JSON (`--json`):
```json
{
  "errors": [...],
  "warnings": [...],
  "summary": {"sections": 12, "errors": 0, "warnings": 1}
}
```

Exit code 0 if no errors; exit code 1 if any error.
