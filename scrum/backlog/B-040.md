## Description

Implement two additional workflow orchestrations to complete the Scrum.md migration to a methodology-agnostic `mdb` workspace orchestrator:
1. **Sequential ID Auto-Increment Scanner**: Compute the next sequential identifier (e.g. `B-040` or `SPR-2026-19`) by searching current section URIs and metadata values matching a given regex.
2. **Workflow Validation (Status Transition Gates)**: Enforce allowed state transitions using rules configured in `.mdb/config.yaml` during `mdb validate`.

## Scope

- **Sequential ID Auto-Increment Scanner (`mdb next-id`)**:
  - Expose CLI command `mdb next-id --prefix <PREFIX> --pattern <REGEX>`.
  - Scan all section URIs and metadata values to extract numbers matching `<REGEX>`.
  - Increment the maximum value found by 1 and return the next formatted ID.
- **Workflow Validation**:
  - Support workflow definitions in `.mdb/config.yaml` declaring allowed statuses and valid state transitions (e.g., `todo -> doing`, `doing -> done`).
  - Integrate transition validation into `mdb validate --root <dir> [--since <ref>]`.
  - Compare current section statuses against historical states (from Git) and report transition violations.
- **Architectural Rationale Documentation**:
  - Document the rationale for placing `.mdb/config.yaml` in the workspace root instead of the memory directory in `README.md`, `specification.md`, and `CONSTITUTION.md` (and the template's `CONSTITUTION.md.j2`).

## Out of Scope

- Implementing custom custom-status UI workflows.
