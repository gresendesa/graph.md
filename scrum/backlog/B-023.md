## Description

Advanced multi-field query against node metadata, extending B-018 (search)
with boolean operators and comparisons. Transforms the graph into a
queryable knowledge base.

## Syntax

```bash
mdgraph query "type=database and environment=prod" --root docs/
mdgraph query "owner=infra or tag=critical" --root docs/
mdgraph query "type=database and environment=prod" --root docs/ --json
```

## Supported operators

- `and`, `or`, `not`
- `=` exact match
- `!=` not equal
- List containment for array fields (tag=infra checks if infra is in tags list)

## Output (--json)

```json
{
  "query": "type=database and environment=prod",
  "results": [
    {"uri": "infra.md#postgres", "metadata": {...}},
    {"uri": "infra.md#redis",    "metadata": {...}}
  ]
}
```

## Acceptance criteria

- [ ] Supports single field `field=value`
- [ ] Supports `and` / `or` boolean combinations
- [ ] Supports `not` negation
- [ ] List containment for array fields
- [ ] --json produces structured output
- [ ] Invalid query syntax returns parse error (exit 1)
- [ ] Tests: single filter, and/or, not, --json
