## Descricao

Implementar o comando principal `mdgraph compose <URI>` (spec section 8.3): o motor entra em modo
transpiler, expandindo recursivamente as diretivas `@include` e produzindo um documento Markdown
unificado.

## Escopo

### Algoritmo de materializacao
- Navegar token a token pela AST da ParsedSection raiz
- Token de texto normal: repassar ao output
- Token `IncludeDirective`: interromper fluxo, buscar AST do no alvo, injetar na cadeia
  - Ajustar nivel matematico dos headings do no incluido (relativo ao nivel do contexto pai)
- Delegacao recursiva ate as folhas (sem ciclos, pois B-008 os rompe)

### Flags
- `--strict`: URI referenciada ausente no index levanta erro fatal (exit code 1).
  Padrao (sem flag): emite Warning no stderr e injeta `<!-- mdgraph:unresolved uri="..." -->` no output.
- `--deduplicate`: se um no ja foi materializado no caminho atual, a segunda ocorrencia e
  substituida por um `@ref` (link simples) em vez de expandida novamente.
  Sem flag: deduplicacao desativada (cada ocorrencia e materializada independentemente).
- `--json`: exporta a arvore composta como JSON estruturado para integracoes de infraestrutura.

## Criterios de aceitacao

- [x] `mdgraph compose raiz.md#secao` retorna documento Markdown unificado no stdout
- [x] Headings de nos incluidos tem nivel ajustado corretamente ao contexto
- [x] URI ausente sem `--strict`: warning no stderr + placeholder HTML no output, sem abort
- [x] URI ausente com `--strict`: erro fatal, exit code 1, sem output parcial
- [x] `--deduplicate`: segundo `@include` para mesmo no vira `@ref` no output
- [x] `--json`: output e JSON valido com estrutura da arvore
- [x] Testes em `tests/test_cli_compose.py` cobrindo todos os casos acima (13/13)

## Riscos

- Alto. E o comando mais complexo: ajuste de niveis de heading, deduplicacao e interacao com
  a deteccao de ciclos (B-008) exigem coordenacao cuidadosa entre modulos.
