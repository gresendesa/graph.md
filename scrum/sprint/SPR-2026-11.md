## Anticipated blockers and mitigations

- Blocker: YAML may parse more than dates into non-JSON-native values.
  Mitigation: Prefer a recursive JSON-safe normalization helper instead of a
  one-off date-only conversion if local code structure supports it.
- Blocker: Fixing only `context` could leave other metadata JSON outputs exposed.
  Mitigation: Inspect related JSON paths and keep the implemented change scoped
  but reusable where appropriate.
- Blocker: Output contract could accidentally change.
  Mitigation: Add regression tests that assert existing schema keys and date
  string value behavior.
