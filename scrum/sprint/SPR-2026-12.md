## Anticipated blockers and mitigations

- Blocker: Git-based `mdb diff` tests may be brittle across environments.
  Mitigation: Use an isolated temporary Git repository fixture and assert only
  stable JSON contract fields.
- Blocker: Query grammar changes may accidentally alter `key~=text` substring
  behavior.
  Mitigation: Keep substring tests and add explicit regression coverage before
  adding `key~=/regex/`.
- Blocker: Structural pseudo-field names may collide with user metadata keys.
  Mitigation: Preserve documented pseudo-field semantics and avoid output shape
  changes unless explicitly versioned.
- Blocker: Documentation can drift from implemented syntax.
  Mitigation: Update `README.md`, `specification.md`, and `llms.txt` in the
  same sprint task as the query implementation.
