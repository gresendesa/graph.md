## Anticipated blockers and mitigations

- Blocker: `index_repository()` is root-oriented and may not provide a direct
  file-only path.
  Mitigation: inspect existing index helpers and choose the smallest compatible
  file-scoped graph construction path.
- Blocker: isolated file validation may report cross-file refs/includes as
  broken.
  Mitigation: document `--file` as isolated validation and keep `--root` as the
  integrated repository validation mode.
- Blocker: `--root` and `--file` together create ambiguous schema path
  resolution.
  Mitigation: reject simultaneous use with a clear error.
- Blocker: schema references in file mode need a deterministic base path.
  Mitigation: define schema resolution relative to the selected file parent for
  `--file`, unless implementation inspection shows a safer root-compatible
  option.
