## Anticipated blockers and mitigations

- Blocker: Legacy memory files may not have clean section boundaries.
  Mitigation: normalize only the useful current records first and preserve old
  content as legacy when needed.
- Blocker: Overly strict schemas could make project memory harder to maintain.
  Mitigation: start with minimal schemas that validate required governance
  fields and allow controlled extension.
- Blocker: `mdb validate --root .` may report unrelated legacy issues.
  Mitigation: document any legacy exception explicitly and keep the sprint
  focused on schema-backed memory normalization.
- Blocker: Adding schemas to consolidator files may conflict with their
  synthetic-summary role.
  Mitigation: keep consolidators concise and schema-backed only where the
  section model adds value.
- Blocker: Too many graph links could make memory noisy or create brittle
  relationships.
  Mitigation: add links only when they connect a clear source-of-truth record
  to a relevant consumer, and prefer `@ref` unless composition requires
  embedding content.
